# The Circuit / Lean Boundary — Map, Amplification, Target, Migration

Status: AUDIT (approve before refactor). Do not treat as landed design.
Scope: the whole circuit-Rust surface (`circuit/`, `circuit-prove/`, and the app/game
consumers `dregg-multiway-tug`, `dregg-automatafl`, `param-compose`, `entity-compose`).
Method: five parallel close-readers classified every layout site; this doc synthesizes
their findings into one executable plan. Grounding spot-checks (below) confirmed the
central claims against HEAD.

---

## 0. The thesis, restated as a boundary law

> **Lean owns everything ABOUT the circuit** — the descriptor (AIR shape), the LAYOUT
> (which column/limb holds what), the witness SCHEMA (where every value goes), the FOLD
> COMPOSITION (which legs connect to what). **Rust owns exactly ONE thing: the
> DENOTATION** — interpreting a Lean-emitted circuit into the plonky3 evaluation and
> doing the raw field/hash arithmetic.

A **byte-pin** is a `#[test]`/`#guard` that reconstructs a Lean value from a hand-written
Rust constant and asserts equality. It is a **GATE, not a DERIVATION**: it catches drift
*after* a human has already written the wrong offset. It cannot stop the human from having
to write it, and it only covers the constants someone remembered to pin. The disease is a
**re-authored mirror** at the circuit-body level: layout Lean already knows, hand-transcribed
into Rust and kept in sync by byte-pins.

### The exemplar (grounded at HEAD)

The fact *"AFTER-block field octet index `i` == field register `r(3+i)`"* is Lean-defined
(`withAfterOctetPins customV3 4`; `weldsAt base+4 ↔ state.FIELD_BASE`). It is hand-encoded
in **three** Rust places that a human must keep reconciled:

- `circuit/src/effect_vm/columns.rs` — `state::FIELD_BASE = 3`
- `circuit/src/effect_vm/trace_rotated.rs:5459` — `pub const CUSTOM_APP_FIELD_ROT_BASE: usize = 4;`
  (doc at :5453 says "Byte-pinned to the Lean `withAfterOctetPins customV3 4`")
- `dregg-multiway-tug/src/fold.rs:384-386` —
  `const APP_FIELD_OCTET_FIRST_REG: usize = 3; let winner_field_key = reg("winner") - 3;`

`winner_field_key` was **guessed wrong three times** (7, then 4), each miss failing a ~70s
fold cycle, because the `- 3` register→octet conversion is a magic number transcribed from a
doc comment. `reg("winner")` itself is *correct* (allocator-derived from schema via
`dregg_schema::allocate_checked`); only the `- 3` is the mirror. This is the whole disease in
one line, and it recurs cross-crate at `circuit-prove/src/ivc_turn_chain.rs:3063`
(`field_k_pi_lo = octet_lo + binding.field_key`).

## 1. THE MAP

### 1.1 The one number that indicts the whole surface

`circuit/src/effect_vm/layout_generated.rs` is `@generated by metatheory/EmitLayoutManifest.lean`
(header, line 1). It is the **cure, already on disk**: 25 layout constants emitted from the Lean
that both defines the geometry and emits the descriptors reading it. Its own header (lines 8-10)
states the stakes verbatim:

> "This module exists because the mirrors drifted, and drift here is not a lint failure — it is
> a soundness bug. The perms/VK completion weld once read limb 37 (`revoked_root` lane-0) while
> the producer wrote limb 38, and every honest setPermissions/setVerificationKey turn was UNSAT."

**Grounded consumer census** (`grep -rn layout_generated --include=*.rs`, whole repo):

| Site | What it does |
|---|---|
| `effect_vm/mod.rs:176` | `pub mod layout_generated;` (declares it) |
| `effect_vm/mod.rs:173` | doc: "Never re-declare one of its constants by hand" |
| `trace_rotated.rs:315` | a comment mentioning it |
| `trace_rotated.rs:6403` | `use ... layout_generated as lean;` **inside the `#[test]`** |

That is the entire consumption. **Production imports it ZERO times.** The Lean-derived answer
exists, is `@generated`, and is quarantined to one mirror-tooth test
(`hand_written_layout_mirrors_the_lean_export`, `trace_rotated.rs:6402`). Meanwhile
`trace_rotated.rs` re-declares **101 `pub const`** and `columns.rs` re-declares **141 `pub const`**,
directly violating the `mod.rs:173` command. `EmitLayoutManifest.lean`'s own header (line 6) even
counts the sin: "Historically Rust re-declared the whole geometry by hand (~89 `pub const`s
mirroring these defs), and the two drifted."

### 1.2 Surface split (synthesized across the five slices)

Rough magnitudes, counting distinct hand-encoded layout facts:

| Class | Count (approx) | Where |
|---|---|---|
| **LAYOUT-MIRROR** (should be Lean-derived) | **~600+** | `columns.rs` (141), `pi.rs` (34), `trace_rotated.rs` (~86 layout consts + 114 `// N` magic-index comments), 15 witness generators (~350), `circuit-prove` fold consts (~13), app/game crates (~30) |
| **DENOTATION** (legit Rust, keep) | thin | `dsl_p3_air.rs` interpreter (~1349 lines, ~95% clean); `descriptor_ir2.rs` `parse/prove/verify_vm_descriptor2`, `check_descriptor2`, `Presence::of`; Poseidon2/field arithmetic; `layout_generated.rs` (the ONE read-from-Lean file, but only ~7% of the surface) |
| **COMPUTATION w/ mirrored placement** | large write-surface | `trace.rs`/`trace_rotated.rs` ~316 `row[...] =` write sites; `descriptor_ir2.rs:8576-8601` witness fill. The *values* (Poseidon2 `hash_many`, 15-bit limb/borrow decomposition) are Rust-legit; the *column each lands on* is a mirror |

The headline: **essentially none of the layout surface is irreducible denotation.** The
genuine Rust is (a) the descriptor-driven AIR interpreter, (b) the field/hash arithmetic, and
(c) the witness *value* computation. Everything about *where a value goes* is Lean-owned layout
hand-transcribed.

### 1.3 The sharpest, most-dangerous hand-encodings (field_key-class bugs waiting to happen)

Ranked by blast radius. Each is a magic offset that, wrong, is either a **silent soundness
hole** (constraint reads an unconstrained neighbor → forgery) or a **fought-for-hours UNSAT**
(honest turn refuses). "Unguarded" = not even covered by the mirror-tooth test.

1. **`fold.rs:384-386` `reg("winner") - 3` + `app_root_pi_offset: 17`** (LAYOUT-MIRROR,
   fought-for-hours, **proven cost**). The exemplar. `17` is a bare literal where the peer
   `circuit-prove/src/joint_turn_recursive.rs:2083` derives the same value as
   `CUSTOM_PI_STATE_PREFIX_LEN`. A wrong `field_key` welds the app-root R to the wrong field lane
   — either UNSAT or a **forged winner passes**.

2. **`trace_rotated.rs:2532-2538` weld offsets `r0@base+1 … r10@base+4+i`** (LAYOUT-MIRROR,
   **unguarded**). The exact twin of the exemplar, at the producer. `base+4+i` is not in
   `layout_generated.rs`. A one-column shift below field-base silently binds the wrong state field.

3. **`trace_rotated.rs:5459` `CUSTOM_APP_FIELD_ROT_BASE = 4`** (LAYOUT-MIRROR, fought-for-hours,
   **unguarded**). The named exemplar constant itself; mis-binds the custom sub-proof app-root R.

4. **`trace_rotated.rs:322-351` the 9 `Felt8Group` completion tables (72 column integers)**
   (LAYOUT-MIRROR, silent-soundness, **7 of 9 groups unguarded**). This is *where the limb-37/38
   REVOKED-ROOT bug physically lived*. `layout_generated.rs` emits only `B_PERMS_COMPLETION=38`
   and `B_VK_COMPLETION=45` (2 of 72 lanes). A completion lane aliasing another group's column ⇒
   two committed roots share a felt ⇒ forging one forges the other. Byte-identical to
   `RotatedLayout.rotated178.groups` in Lean, which **has a proven-disjoint `groupTable` projection
   that is NOT wired to the emit** (`RotatedLayout.lean:119-121`: "today emits the scalar constants
   … but NOT the group position lists — those are still hand-duplicated").

5. **`circuit-prove/src/joint_turn_recursive.rs:105` `CUSTOM_COMMIT_PI_LO = 46`** (LAYOUT-MIRROR,
   silent-soundness). The primary fold-connect index for the custom leg; `customPiExposure` in
   `EffectVmEmitRotationV3.lean:2117` literally *computes* `46+k`. The whole custom+dsl+bridge fold
   family (`dsl_leaf_adapter`, `mpt_holding_leaf`, `carrier_pin_twin`) re-cites `PI 46..53`.

6. **`joint_turn_recursive.rs:941` keystone app-root connect** (LAYOUT-MIRROR, silent-soundness).
   THE keystone weld ("published R == cell real field"). Wrong lane ⇒ a forged app root passes.

7. **`trace_rotated.rs:290-301` carrier octet bases `89/97/105`** (LAYOUT-MIRROR, silent-soundness,
   **unguarded**). Manually `+1`-shifted on the REVOKED-ROOT flag day (were `88/96/104`); absorbed
   into `state_commit` and PI-exposed. A human already moved these three by hand once.

8. **`caveat_admission_leaf_adapter.rs:108-124` operand slot map `OP_* 0..7`** (LAYOUT-MIRROR,
   silent-soundness). A swapped operand index compares the wrong pair (e.g. `req_time` vs
   `cav_budget`) and the AIR is satisfiable over the mislabeled columns.

9. **`descriptor_ir2.rs:1383-1385` Transition bound `EFFECTVM_STATE_BEFORE_BASE + hi`**
   (LAYOUT-MIRROR, one leak into otherwise-clean denotation). The descriptor supplies relative
   `hi/lo` but Rust adds a hand base; the descriptor should name the absolute column.

10. **`automatafl/src/air.rs:710-758` + `moves.rs:528,842,888,1118` bare `16` door prefix**
    (LAYOUT-MIRROR, silent-soundness). `PI[0..16)` prefix copied as a raw literal at ~9 sites;
    a reorder of `add_pi`/`bind_pi` calls silently misbinds.

## 2. THE AMPLIFICATION — how the mistake spread

Git history is unreliable (the AGPL relicense rewrote every hash), so this is the **structural**
reconstruction, not a dated one. The evidence is in the code's own layering.

### 2.1 The seed pattern

The original, defensible move: **hand-write one AIR generator, then guard it with a byte-pin
against the Lean descriptor it must match.** For a single descriptor this feels safe — the pin is
green, the number matches, drift will fail CI. `EmitByName.lean` even states the correct law
("Rust interprets; Rust authors nothing") and the descriptor-JSON path (`include_str!` + `parse_vm_descriptor2`, `effect_vm_descriptors.rs:79+`) *honors* it: the constraint bodies and
relative column indices are genuinely Lean-authored and Rust-interpreted. That path is clean.

### 2.2 The amplifier: copy-per-descriptor

The mistake was to **copy the generator+byte-pin idiom per new descriptor member** instead of
extracting the layout into the emitted artifact. Concretely the same "column-mirror block + a
`builder_matches_lean_golden` test" idiom is instantiated **15 times** across the witness
generators (`attested_fact`, `blinded_membership`, `note_spending`, `predicate_comparison`,
`non_revocation`, `predicate_arith`, …), each re-authoring its own private name→index map (~350
constants total), each closed by its own byte-pin. The rotated EffectVM geometry is declared in
**three** files (`layout_generated.rs` + `trace_rotated.rs` + `columns.rs::rotation`);
`EFFECT_VM_WIDTH=188` appears in **five**; the field-register base `3/4` in **three**.

### 2.3 Why byte-pins made it feel safe while hiding that it is a mirror

- **A pin is green when the copies agree — it says nothing about whether the copy should exist.**
  The gate proves "Rust == Rust-reconstruction-of-Lean" at test time; it never proves "Rust reads
  Lean." Green feels like verification; it is only *drift detection on the constants someone
  remembered to pin*.
- **Coverage is a fraction and invisible.** The mirror-tooth test cross-checks ~20-26 of ~81
  `trace_rotated` constants; the other ~60 carry only doc-comment provenance ("Lean twin …"). A
  drift in an unguarded constant ships green.
- **Pins run only on `cargo test`, per member.** A wrong offset PROVES fine on every descriptor
  member you did not exercise (the limb-37/38 bug proved fine on everything except setPermissions/
  setVK).
- **The doc comments themselves rot** — `trace_rotated.rs` absolute-column comments `// 187 / 186 /
  237` have already drifted from the real `188 / 427` (`EFFECT_VM_WIDTH=188`,
  `AFTER_BASE=188+239=427`), proving humans hand-track absolute columns and the tracking rots.

### 2.4 The tell: the cure was half-built and then stranded

`EmitLayoutManifest.lean` was written *specifically to kill this class* — its header enumerates the
three real soundness/liveness bugs it was built to prevent (REVOKED-ROOT +1 shift; `B_SPAN 227→239`
audit-col rot; `vaultSat` span growth). It emits `layout_generated.rs` correctly. But the wiring
stopped at a test. The infra reached ~60% of the way to the target and the last step — *making
production read the emit instead of the hand copy* — was never taken. So the amplification is not
"nobody knew"; it is "the fix exists, is proven, and is unconsumed." `RotatedLayout.lean:119-121`
names the remaining gap in the same words as this thesis.

## 3. THE TARGET ARCHITECTURE — where the line SHOULD be

### 3.1 What Rust keeps (DENOTATION + value-COMPUTATION)

- The descriptor-driven AIR interpreter: `dsl_p3_air.rs` `eval_expr`/`eval` (reads every column
  index from the Lean `ConstraintExpr`), lowering to `p3 AB::Expr`. **Keep as-is** — this is the
  correct boundary.
- `descriptor_ir2.rs`: `parse_vm_descriptor2`, `check_descriptor2 → MainLayout`, `Presence::of`,
  `prove/verify_vm_descriptor2`. Table presence and layout are *derived from the descriptor*. Keep.
- Poseidon2 permutation + round arithmetization, `hash_many`, field ops, the p3-internal witness
  columns (Poseidon2 intermediates, IsZero inverse witnesses) — Lean need not know Poseidon2's
  arithmetization. Keep.
- Witness **value** computation: the 15-bit limb/borrow-bit decomposition, the digest chaining
  values. Keep the arithmetic; only the *placement* moves to Lean.

### 3.2 What Lean emits (four artifacts, three already partly exist)

Lean already emits the **constraint bodies** (descriptor JSON — denotation-correct, consumed) and
the **scalar spine** (`layout_generated.rs` — emitted, not wired). It must additionally emit, as
`@generated` Rust and/or `include_str!`'d data parsed once (the pattern `descriptor_by_name.rs`
already uses):

1. **The full COLUMN/OFFSET MANIFEST.** Extend `EmitLayoutManifest.lean` past its current 25
   scalars to the whole geometry: the v1 face (`sel::*` 30 consts, `state::*` incl `FIELD_BASE=3`,
   `PARAM_BASE`, `AUX_BASE`, `STATE_*_BASE`), the rotated bases (`BEFORE_BASE`, `AFTER_BASE`,
   `CAVEAT_BASE`, `CUSTOM_HOST_WIDTH`, `N_ROT_SITES`, `GRAD_ROT_WIDTH`), and the octet bases
   `[89,97,105]`. Rust replaces every `pub const` with `pub use layout_generated::*`.

2. **The GROUP-POSITION TABLE** — the missing piece `RotatedLayout.lean:119-121` names. Render
   `RotatedLayout.rotated178.groupTable` as `pub const ALL_FELT8_GROUPS: [[usize;8];9]`. Because
   `groupTable` is a projection of a proven-`Legal` layout (`rotated178_legal`: disjoint + inBounds
   + bodyAligned by `native_decide`; `rotated178_groupTable_columns = 80`), the emitted table is
   **disjoint-by-construction** — the aliasing soundness hole becomes unrepresentable. This kills
   the 72 hand integers at `trace_rotated.rs:322-351`.

3. **The WITNESS-SCHEMA / named column table.** The 231 `PARAM_BASE + N` weld sites already carry
   Lean names in doc comments (`FLOOR0_COL`, `B0_COL`, `INV0_COL`, `WIT_DIGEST_COL`, `CHIP_LANE[j]`,
   `AUTH_DIGEST_COL`). Emit them as named constants so each weld reads `lean::FLOOR0_COL` instead of
   computing `PARAM_BASE + 5 + k`.

4. **The PI-EXPOSURE / FOLD-COMPOSITION table.** The objects
   `customPiExposure` / `customFieldKExposure` / `rotateV3` / `factoryV3Carriers` /
   `withAfterOctetPins` already *compute* the PI slots (`46+k`, `50+k`, `54+k`, field-octet,
   wide-anchors) inside Lean when they build `piBinding` constraints — they just aren't serialized.
   Emit a per-descriptor-member `PiLayout { commitment=(46,8), vk=(54,4), state_prefix=(0,16),
   app_field_octet=(n-24,8), wide_anchor_old=(n-16,8), wide_anchor_new=(n-8,8), … }` plus a
   `SegmentLayout` for the fold ABI. Then `CUSTOM_COMMIT_PI_LO`, `CUSTOM_COMMIT_LEN`,
   `BRIDGE_TUPLE_LEN`, `octet_lo/old_first`, and the connect loops all become layout reads;
   `fold.rs` asks `layout.app_pi("winner")` (=17) and `layout.octet_index_of_register(reg("winner"))`
   instead of `- 3`.

5. **(supporting) The POSEIDON2 DOMAIN-SEPARATION TABLE.** The ~6 tag constants
   (`0xFACF`, arity `2/4/3`, `CAP_FACT_MARK`, capacity-lane positions) are written **three** times
   (`dsl_p3_air.rs` symbolic twin at :322-359, its concrete twin at :394-448, and `poseidon2.rs`).
   Emit one per-hash-form input-state table so all three read one source. (Judgment call: this is a
   hash-schedule fact, arguably denotation-adjacent — but it is currently a hand-agreeing triple, so
   a single emitted table is strictly better.)

### 3.3 The consumption shape (already proven in-tree)

Two existing patterns show the target already works and just stop at the Rust boundary:

- `dregg_schema::allocate_checked` resolves `reg("winner")` **by name** from the schema — no magic
  register index. Extend it to resolve octet index too.
- `param-compose/src/pi.rs:40-76` derives *every* PI index from one imported `APP_BASE =
  CUSTOM_PI_STATE_PREFIX_LEN` + `DIGEST_FELTS` — no magic `16/17/21`. The residual is only that the
  base is a Rust constant that is itself a hand-mirror of Lean; point it at the emitted table.

**End state:** DENOTATION = `parse_vm_descriptor2` + Poseidon2/field arithmetic + value
computation. Everything else Rust READS from generated modules. The ~26 byte-pins and the
drift-guard tests become vacuous (one source cannot drift).

## 4. THE MIGRATION PLAN — ranked, incremental, each step independently landable

Each step is landable and verifiable alone. **Keep every existing byte-pin as a safety net until
the step that makes it vacuous lands, then delete it in the same commit.** Order is
highest-value-lowest-cost first — the field_key class is the cheapest win and kills a whole
bug family.

### Step 1 — Wire the scalar spine that already exists (LOW cost, HIGH value)

`layout_generated.rs` is `@generated` and unconsumed. In `trace_rotated.rs` and `columns.rs`,
delete the hand `B_*` / spine `pub const`s and `pub use crate::effect_vm::layout_generated::*`.
- **Verify:** full `cargo build -p dregg-circuit` compiles (single declaration); the mirror-tooth
  test `hand_written_layout_mirrors_the_lean_export` becomes vacuous → delete it in the same commit.
- **Net:** ~25 mirrored constants → 0; three declaration sites → one. Directly satisfies the
  `mod.rs:173` command the code currently violates.

### Step 2 — Emit + read the octet→register map, killing the field_key class (LOW cost, HIGHEST value)

Add to the manifest: `state::FIELD_BASE`, `CUSTOM_APP_FIELD_ROT_BASE`, and an explicit
`app_field_octet` array with a `octet_index_of_register(reg) -> usize` helper. Switch the three
exemplar consumers:
- `trace_rotated.rs:5459` → read `lean::CUSTOM_APP_FIELD_ROT_BASE`.
- `trace_rotated.rs:2536` weld loop → read the emitted base.
- `fold.rs:384-386` → `layout.octet_index_of_register(reg("winner"))`, kill `- 3`; and
  `app_root_pi_offset: layout.app_pi("winner")` instead of literal `17`.
- `ivc_turn_chain.rs:3063` → read `field_k_pi_lo` from the PiLayout.
- **Verify:** run the ~70s custom/tug fold cycle end-to-end (the one that cost hours) — it must
  prove first try. This is the exemplar retired.
- **Net:** the entire `field_key`-guess class is a Lean query; no human writes the offset again.

### Step 3 — Emit the proven group table, killing the completion-lane soundness hole (MED cost, HIGH value)

Wire `RotatedLayout.groupTable` through `EmitLayoutManifest` → `ALL_FELT8_GROUPS: [[usize;8];9]`
(+ octet bases `[89,97,105]`). Replace `trace_rotated.rs:322-351`. Because the source is
proven-disjoint, the aliasing forgery becomes unrepresentable.
- **Verify:** `cargo test` including the perms/VK setPermissions/setVK prove-path (the historic
  bug); the mutation canary that pokes a completion lane must still refuse.
- **Net:** 72 hand integers → 0; the limb-37/38 bug class structurally closed.

### Step 4 — Parse the staged manifest JSONs instead of pinning them (MED cost, MED value)

`rotation-layout-v3-staged.json` and `rotation-caveat-layout-v3-staged.json` already exist as Lean
output. Parse them (tiny serde struct, the descriptor-JSON pattern) into `columns::rotation` /
`columns::caveat` at build time. Delete `rotation_layout_matches_lean`
(`effect_vm_descriptors.rs:2141`) and `rotation_caveat_layout_matches_lean` (:2298) — the parse
*is* the check.
- **Verify:** parse round-trips to the current constants; full build green.

### Step 5 — Emit the PiLayout / SegmentLayout for the fold family (MED-HIGH cost, HIGH value)

Serialize `customPiExposure`/`customFieldKExposure`/`rotateV3` into a per-member `PiLayout`
(`include!`'d like the staged-registry TSVs). Switch `joint_turn_recursive.rs` (`CUSTOM_COMMIT_PI_LO=46`,
`CUSTOM_COMMIT_LEN`, `BRIDGE_TUPLE_LEN`, connect loops), `ivc_turn_chain.rs` (`old_first/new_first`
tail-anchor derivation), and the `dsl/mpt/carrier` leaf adapters that re-cite `PI 46..53`.
- **Verify:** the custom + bridge + dsl fold family all prove; delete the `#guard CUSTOM_COMMIT_TEETH_COL`
  cross-pins as they go vacuous.

### Step 6 — Emit the witness-schema named columns for the 231 weld sites (HIGH cost, MED value)

Emit `FLOOR0_COL`, `B0_COL`, `INV0_COL`, `CHIP_LANE[j]`, `AUTH_DIGEST_COL`, … so the 9 `*_weld.rs`
files read named columns instead of `PARAM_BASE + N`. Largest mechanical surface; do it weld-file
by weld-file, each independently verifiable against its mutation canary.

### Step 7 — Generalize to the 15 witness generators (HIGH cost, MED value, repetitive)

Extend the emitted descriptor (`EffectVmDescriptor2`, `descriptor_ir2.rs:676`) with a
`columns: Vec<(String,usize)>` + `pi_slots: Vec<(String,usize)>` symbol table (the Lean `*Emit.lean`
already names these internally). Collapse each generator's private block to `desc.col("FACT_HASH")` /
`desc.pi("ROOT_PI")`. Delete each `builder_matches_lean_golden` as it goes vacuous. This is the long
tail — safe to fan across agents once the emit surface (Steps 1-5) is in place, since it is
application of a proven pattern, not research.

### Step 8 — Emit the v1 sel/state face + Poseidon2 domain table (LOW-MED cost, closes the residue)

`columns_ir1.rs` carrying `sel::*` / `state::*` / `*_BASE`, so `SELECTOR_DESCRIPTORS`
(`effect_vm_descriptors.rs:399-556`) is built by reading each descriptor's own `selectorGate`; and
the Poseidon2 domain-separation table so the three hand-agreeing hash-tag copies collapse to one.

### Safety-net discipline during migration

- A byte-pin is **deleted only in the commit that makes it vacuous** (the hand copy it guards is
  gone). Never delete a pin while two copies still exist.
- After any step that touches a shared layout module, build the **whole** workspace, not just the
  changed crate — a deepened emitted struct can red a downstream consumer invisibly
  (`descriptor_ir2` + all consumers).
- Steps 1-3 are the approve-and-go core (they retire the exemplar and the historic soundness bug).
  4-8 are the tail; each is independently landable, so partial completion is still net-positive.

## 5. HONEST SCOPE

### 5.1 Genuinely stays Rust (not a mirror, do not touch)

- The p3 AIR interpreter (`dsl_p3_air.rs eval_expr/eval`) — reads indices from the descriptor.
- `descriptor_ir2.rs` prove/verify/check/`Presence::of` — derives everything from the descriptor.
- Poseidon2 arithmetization + p3-internal witness columns (round intermediates, IsZero inverses).
- Witness **value** arithmetic (limb/borrow decomposition, digest chaining). Lean cannot
  auto-derive a satisfying witness — that is genuine computation. Only the *placement* moves.

### 5.2 Genuinely hard / not a quick win

- **Witness computation cannot be auto-derived from layout.** Steps 6-7 move *placement* to Lean
  but the value arithmetic stays hand-written; the win is narrower than "delete the generator."
- **The Poseidon2 domain table (Step 8 half)** is a judgment call — it is hash-schedule, arguably
  denotation. Emitting it is an improvement (kills the triple copy) but not a soundness necessity.
- **`entity-compose OUTCOME_WIDE_BASE=64`** and similar are *free app placements*, not Lean-owned
  layout — their collision-safety is a hand-argued invariant, not a mirror. Leave as computation;
  do not manufacture a Lean fact that does not exist.

### 5.3 False hopes to avoid

- **"Delete the byte-pins now."** No — a pin is a safety net until its hand copy is gone. Deleting
  pins before Step N lands *removes drift detection while the mirror still exists*.
- **"One giant emit-everything commit."** No — the emit struct changes are shared-interface changes;
  batching them makes the red umbrella invisible. Ship per Step, build the whole tree each time.
- **"The descriptor JSON already covers it."** It covers constraint *bodies* and *relative* indices;
  it deliberately drops the *symbol table* (names → absolute columns). That gap is exactly what
  Steps 1, 5, 7 add.

### 5.4 Rough size estimate

- **Emit side (Lean):** `EmitLayoutManifest.lean` grows from ~25 items to the full manifest + group
  table + PiLayout + witness symbol table; plus serializing the PI-exposure objects that already
  exist. Estimate ~4-6 Lean emit additions, all backed by layouts Lean already proves
